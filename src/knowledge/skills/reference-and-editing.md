# 参考图理解与图片编辑

## 一、参考图 / 附件角色判断

附件是正式需求来源，不是装饰。先判断角色，再提取可迁移信息：
- **logo/品牌资产**：提取品牌色、字标比例、图形元素；确认落在顶部/主墙/接待台/LED·KV/导视。
- **产品图**：提取体量、材质、陈列密度、演示需求、安全距离。
- **参考效果图**：提取空间骨架、构图、材质、灯光、主次关系、可迁移语言——**不等于必须复刻**。
- **平面图/场地图**：识别开口、邻展、主通道、柱位、入口、限高、消防/承重限制。
- **施工/规范截图**：识别限高、吊点、消防、承重、搭建限制。

## 二、有序参考图硬协议

- 上传/拖入顺序是硬协议：用户说"第一张/图1/第 N 张"时绑定到本轮参考图序号，**不按文件名、视觉相似度或自动判断重排**。
- 用户显式指定的角色优先于自动判断（"第二张是甲方 logo，第三张是电视支架"）。
- prompt 写成 ordered contract：`Reference image #N = [用户指定角色]`，说明落位/比例/主次/是否占位或后期贴图。
- 参考图是 logo/文字/KV：控制落位、比例、主次、发光/贴图方式，不要求模型精确复刻小字。
- 参考图是支架/产品/结构件：写清放进哪个功能区、与墙/屏/展柜/地台如何连接、比例是否缩放。
- 多张参考图有主次：用户指定用途优先；未指定的只作风格/材质/结构辅助，不覆盖已确认方案。
- 涂抹后的 inpaint 预览要理解为"原图 + 黑白蒙版"，不要把彩色涂抹当成要生成的视觉元素。

## 三、附件证据抽取

1. 识别角色（文件名/格式/摘要/图像内容）。
2. 抽硬约束（尺寸/开口/限高/吊点/预算/功能/产品尺寸/品牌规范）。
3. 抽设计线索（颜色/材质/光影/构图/主视觉/情绪，转译为可执行设计语言，不只写"高级/科技/好看"）。
4. 标不确定项（来源不明、互相冲突、扫描不清、只看到部分页）→ 写入风险，不当已确认事实。
5. 附件已提供的信息不要重复泛问，直接进入确认或只追问缺口；附件用途不明时用问题卡确认角色。
6. 附件与用户文字冲突时，以用户最新明确文字为准，同时提示冲突。

## 四、语义编辑（revise_asset 的默认路径）

把模糊中文意图收敛成一条**精确、保守、可执行**的英文编辑指令：
- 只描述"要改什么、不要改什么、保持什么"。`Using the provided image, change only: [...]`。
- **优先保持**原图结构、比例、品牌位置、材质、灯光、相机角度；没被明确要求改的一律保持不变。
- 模糊词（"高级点/科技点/通透点"）必须落到材质、灯光、色彩、局部体块或陈列方式上。
- **局部修改优先于整体重构**；指令要短、准、稳，避免重新设计整张图。
- 若修改会影响遮挡/动线/品牌可见性，显式写进说明。
- 若用户本质是"重做方向"而非"编辑"，先按最保守局部修改解释（除非上层已明确走重做链路）。

## 五、蒙版编辑（局部精修）

- 白色=要改的区域，黑色=PIXEL-PERFECT 保持与原图完全一致。
- 用户文字只指导"白色区域改成什么"，不涉及蒙版外任何部分。
- 修改区边缘必须无缝融合：匹配光照方向、色温、材质纹理、透视。
- 输出必须与原图**尺寸、宽高比完全一致，不裁切不缩放**。
- 描述模糊时做最保守的局部替换，不扩大范围。
- 新内容物理合理：支撑结构、遮挡关系、地面接触必须正确。
- 双图迁移：源图白区=要借用的内容，目标白区=要被替换/融合的位置；按参考图顺序理解源/目标，匹配目标图透视/尺度/光照/遮挡/材质。
