# 提问完备度 Rubric

**这是判断工具，不是台词。** 每轮用户说完，大脑对照本表做 gap 分析：哪些关键字段还空着、哪些答案触发了新的必问项、能用默认值填的就填（并告知）、不能默认且会改方案方向的才问。一次最多 3 问，每问 2-4 个具体可视化选项。**核心原则：信息不够就问，够了先确认再出方案，绝不捏造用户没说过的关键信息。**

## 一、完备度字段词典（Blocking 级 —— 不确认或未显式默认前不得出完整方案）

字段 key 可直接作强类型 Brief 的主键。

### 空间硬约束
- `footprint` 面积（长×宽，**保留长/短边语义**）— 36㎡ 和 108㎡ 方案完全不同。
- `opening_sides` 开口方式（几面开）— 决定墙体数量、展示面、动线。
- `opening_wall_orientation` 开口方位（背墙/开放边靠**长边还是短边**）— 6×12 三面开背墙靠长边 vs 短边骨架完全不同。
- `two_side_opening_relationship` 两面开是**相邻直角**还是**相对平行**。
- `main_aisle_orientation` 主通道方向（多开放边时定品牌主视觉/接待台/首图视角）。
- `height_limit` / `height_wall_alignment` 限高与高度口径（"6m"是否含 Truss；会议室是否与主墙齐平）— 把含 Truss 总高误当板墙高会导致比例错误。
- `top_structure_strategy` 顶部/Truss 策略（无顶/门头/落地/吊装）— 每个展台都要处理天际线、挂点、远距识别。
- `truss_center_form` 顶部中部造型（空心/环形/灯框/灯阵/软膜/几何/场景化）— 不确认会顶部空洞或裸架过重。

### 核心目的与内容
- `purpose` 核心目的（品牌形象/产品展示/新品发布/招商洽谈/体验互动）— 各绑不同功能区。
- `product_layout_display` 重点展品 + 产品摆放与动线（集中/分散/入口爆品/演示优先）。
- `brand` 品牌/公司（强品牌色如华为红/索尼黑会锁定色彩）。
- `brand_asset_placement` 品牌名称、slogan、Logo/关键图标 + 落位方式 — 不确认方案只有"品牌区"空壳（红线见 `brand-assets`）。

### 视觉方向
- `style_tone` 整体调性/风格方向。
- `color_scheme` 主色+辅色（每个最终 prompt 都要明确主辅色关系）。

每个字段都附"为什么必问 / 不确认会怎样"——提问时把后果讲给用户（见问题设计）。

### 项目基础与输出偏好（intake 早期顺带收集，影响交付而非空间骨架）
- `project_meta` 项目基础信息：展会名称/地点/时间、品牌/公司、行业、交稿或搭建时间（deadline 影响方案复杂度与可落地性）。
- `output_preference` 输出偏好：偏概念提案 / 偏真实商业效果图 / 偏稳妥落地 / 偏视觉冲击。**这条直接影响生图的"分量"**——偏概念→单张快出；偏最终交付→best-of-N 择优 + 允许修复（见 `rubrics/inspection`）。

## 二、条件触发清单（依赖式提问 —— 某答案触发联动才追问）

| 触发条件 | 追问 | 原因 |
|---|---|---|
| 面积≤36㎡ + 多功能 | 功能区优先级排序 | 空间紧张，不能全要 |
| 提到 LED 主屏 | 尺寸+安装方式 | 内嵌墙体厚~80cm，吃空间；吊装走顶部逻辑 |
| 提到洽谈区/VIP | 封闭程度+人数 | 封闭外墙≥2.5m，形成大体块 |
| 封闭会议室贴主墙/LED墙 | 是否与主体板墙齐平 | 齐平 4.0-4.4m vs 矮隔断 2.5-2.8m |
| 说"6m 高" | 是否含 Truss | 常规 6m 多指含 Truss 总高 |
| 选 Truss 顶部 | 支撑方式（柱撑 vs 吊装） | 无吊点不得写吊装 |
| 大跨度门头/大屏 | （后台自动套跨度安全，不问用户） | 木≤6m、桁架≤9m |
| 四面开口 | 展示位安排 | 无邻展强制墙 |
| 多台电视/屏幕 | 数量+分布（集中 vs 分散） | 影响墙体设计 |
| 双层展台 | 层高分配 | 总高~6m，首层~3m |

## 三、行业默认值表（可自动填充，但须 disclosure）

| 条目 | 默认值 | 依据 |
|---|---|---|
| 地面 | 地台10cm+地板/地毯 | 藏电子设备线 |
| 主体板墙高 | 国内~4.4m / 国外~4.0m | **非含 Truss 总高** |
| 含 Truss 总高 | ~5-6m | 含 Truss/吊牌/灯架/门头 |
| 接待台 | 90cm 高、深45cm | 人体工学 |
| 展柜 | 75-100cm 高 | 视线高度 |
| 墙体厚 | ~10cm；LED 墙 ~80cm | 含检修通道 |
| 洽谈区外墙 | 半封闭2.5-2.8m；贴主墙则齐平4.0-4.4m | 隔音/立面连续 |
| 灯光色温 | 冷白5000-5500K(科技)/暖白3000-3500K(商务) | 行业惯例 |
| 画面视角 | three-quarter wide-angle | 商业效果图标准 |

## 四、每轮 gap 分析循环

1. 用户刚说了什么？更新了哪些字段？
2. 查"完备度字段词典"→ 哪些还空？
3. 单独检查 开口方位、顶部/Truss 策略、产品摆放与动线 → 即使没提也必须问或显式默认。
4. 已选顶部结构 → 检查中部造型是否明确，未明确二次追问。
5. 检查品牌：名称/slogan/Logo/关键图标/落位是否明确 → 未明确则确认或显式占位。
6. 查"条件触发清单"→ 用户回答触发了哪些新必问项？
7. 内部做方案前审查（八维，见下）→ 还有 blocking 缺口吗？
8. 空的必问项：能默认的→填+告知；不能默认且会改方向的→必须问。
9. 必问项=0 → 出信息确认（让用户看当前信息/默认/缺失/风险）；必问项>0 → 问最关键的 2-3 个。

## 五、方案前审查（内部，只把 blocking 缺口转成问题，不输出评审报告）

优先级固定：**结构安全 > 面积开口方位限高 > 高度口径 > 顶部/Truss > 产品与动线 > 品牌资产落位 > 材料灯光 > 品牌表达 > 生图一致性。** 八维：空间硬约束 / 高度口径 / 顶部Truss / 跨度结构 / 产品展示 / 动线功能 / 材料灯光 / 品牌表达 / 生图一致性。算法：逐项标 pass/defaultable/blocking/risk → 合并同源 → 按优先级取≤3 → 一问只解决一个字段。

## 六、问题设计（选项必须具体、可视化、说明代价）

每个问题带：`rationale`（为什么此刻问，引用已知缺口）、`dependsOn`（依赖哪些已知信息）、`blockingField`（补哪个字段）；每个选项带 `designImpact`（选了会怎样）+ `physicalBasis`（结构/动线/尺度/预算依据）。

✅ 风格（具体+常识类比）：A. 暗场+重点照明+金属玻璃（像索尼/奔驰展台）B. 白色+柔和漫射光+几何体块（像苹果零售店）C. 暖色木饰面+柔光（像五星酒店大堂）D. 通透玻璃+开放流线（像高端办公）
✅ 面积（常识锚定）：A. 小型（6×6m，约一间教室）B. 中型（9×6m，约一个大客厅）C. 大型（12×9m，约半个篮球场）
✅ 洽谈区（说明空间代价）：A. 4人紧凑（一桌四椅，约占1/12，半封闭）B. 6-8人标准（约1/8）C. 10人+（约1/6，需全封闭）D. 不需要
❌ "你喜欢什么风格？" / "A.科技感 B.简约 C.高端"（太空泛）。
顶部不能只问"要不要 Truss"，必须给空间后果；中部造型不能漏问，必须给美感后果；品牌不能只问"要不要 LOGO"，必须问内容+落位。

## 七、自然语言 → 字段提取

| 用户说 | 提取 |
|---|---|
| "6乘3的展台" | 18㎡，6×3 |
| "标准展位" | 3×3=9㎡，一面开 |
| "岛型/中岛" | 四面开 |
| "角位" | 两面开 |
| "三面开"+长方形 | 追问背墙长边/短边 |
| "两面开" | 追问相邻 vs 相对 |
| "6米高" | 先理解为含 Truss 总高，板墙仍 4.4/4.0 |
| "像苹果店那种" | 极简白+漫射光+几何体块（调性方向） |
| "要能谈生意" | 需洽谈区（封闭度待定） |
| "预算有限" | 避免复杂顶部/双层/大 LED，优先木结构+经济材质 |

## 八、"够不够出方案"终检（Prompt 10 信息块）

1-10 必须到位才算信息足以写 DesignSpec：① 空间类型与展台属性 ② 高度体系 ③ 顶部结构 ④ 顶部中部造型 ⑤ 主体分区(3-4功能区) ⑥ 产品摆放与动线 ⑦ 品牌资产与落位 ⑧ 主视觉面 ⑨ 风格定性 ⑩ 色彩关系。⑪ 材料/灯光/镜头可自动填充；跨度安全永远后台自校正。
